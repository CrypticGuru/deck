REST
(Representational State Transfer)

Overview

A resource is an object with a type, associated data, relationships to other resources, and a set of methods that operate on it.

It is similar to an object instance in an object-oriented programming language, with the important difference that only a few standard methods are defined for the resource.

Some Benefits of an API:

  • Separates user interface from data model
  • Improves scalability
  • Facilitates caching

HTTP Response Codes

There are many HTTP reponse codes, and they cover most situations. Our APIs should return relevant HTTP status codes.

For example:

  • When a resource is successfully created, the API should return a 201. The response could also contain a Location header with a link to the newly created resource.
  • If a request is made for a resource that doesn't exist, a 404 should be returned.
  • If a request is made to create a new resource which already exists, a 409 should be returned.

Use of the HTTP response codes (instead of returning JSON) can speed up requests and reduce bandwidth costs.

HTTP Verbs

The four primary HTTP verbs are:

GET

The GET method is used only to view a resource, and not change it.

PUT

The PUT method is used to update a resource.

The request should contain the updated representation of the original resource.

POST

The POST method is used for the creation of a new resource.

The request should contain the required information for the API to create a new resource.

DELETE

The DELETE method is used to delete a resource.

Examples

GET

http://www.securitymetrics.com/api/customers/1234
would return a representation of the customer who's ID is 1234.

http://www.securitymetrics.com/api/customers/1234/products
would return a representation of the customer's products who's ID is 1234.

PUT

http://www.securitymetrics.com/api/customers/1234
would update the customer who's ID is 1234.

http://www.securitymetrics.com/api/customers/1234/products/5678
would update the product by ID belonging to this customer.

POST

http://www.securitymetrics.com/api/customers
would create a new customer.

http://www.securitymetrics.com/api/customers/1234/products
would create a new product for the customer.

DELETE

http://www.securitymetrics.com/api/customers/1234
would delete the customer who's ID is 1234.

http://www.securitymetrics.com/api/customers/1234/products/5678
would delete the customer's product who's ID is 5678.
